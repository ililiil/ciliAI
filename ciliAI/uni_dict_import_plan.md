# 字典与院校维护 Excel 导入准备方案

为了让客户系统性地准备底层数据（字典、国内院校、国外院校），我们将其分为 **三张独立的 Excel 表**。

其中，**院校涉及图片的列**采用“指定图片文件全名”的设计，导入时由系统自动提取文件名去关联已上传的图片库。

---

## 第一部分：字典底层数据 (建议分为 5 个 Sheet 页)

为了让整理更清晰，底层字典不建议混在一起，而是按照“字段结构”分为以下 5 个 Sheet。

### 1. 通用标签字典表（适用类别最多）
> **包含字典**：国内外院校推荐标签、项目标签、项目推荐标签、项目亮点、入学要求、海外院校标签、国内院校认证标签、国内优势学科、国家列表、中国省份、海外州省。

| 列号 | Excel 表头名称 | 必填 | 给客户的填写说明 |
| :--- | :--- | :--- | :--- |
| **A** | **所属字典类别** | 必填 | 从上面列出的类别中选一个填写，例如：`国内院校认证标签`。 |
| **B** | **标签名称** | 必填 | 例如：`985高校`、`双一流`、`历史悠久`。 |
| **C** | **排序号** | 选填 | 填写数字，用于控制下拉框的显示顺序。 |

---

### 2. 海外院校实力字典表
> **包含字典**：海外院校实力（带有副标题属性）

| 列号 | Excel 表头名称 | 必填 | 给客户的填写说明 |

用于准备中国国内合作大学的信息。

| 列号 | Excel 表头名称 | 必填 | 给客户的填写说明 |
| :--- | :--- | :--- | :--- |
| **A** | **国内院校名称** | 必填 | 院校完整中文名，例：`南京大学`。 |
| **B** | **院校外文名** | 选填 | 院校标准英文全名，例：`Nanjing University`。 |
| **C** | **所在省份** | 必填 | 必须和系统里“中国省份”字典一致，例：`江苏`。 |
| **D** | **优势学科** | 选填 | 从字典选，多个用**英文逗号**隔开，例：`工学, 管理学`。 |
| **E** | **院校认证标签** | 选填 | 从字典选（如985, 211），多个用**英文逗号**隔开，例：`985高校, 双一流`。 |
| **F** | **推荐标签** | 选填 | 从“国内外院校推荐标签”选填，例：`历史悠久, 排名第一`。 |
| **G** | **院校介绍** | 选填 | 院校详细介绍，支持多段落，换行请按 `Alt+Enter`。 |
| **H** | **Logo 图片名** | 选填 | 请填完整文件名加后缀。命名规范为 `[院校名称]_logo.png`。例：`南京大学_logo.png`。 |
| **I** | **背景大图名** | 选填 | 请填完整文件名加后缀。命名规范为 `[院校名称]_bg.jpg`。例：`南京大学_bg.jpg`。 |

## 4. 图片文件（Logo/背景库）命名与导入规范

除了提报上方的 Excel 表格数据，客户还需要配合提供一个 **【院校图片包 (ZIP)】**：

1. **命名规范要求：**
   - 强烈建议图片以 **院校中文全称** 作为前缀，方便肉眼核实。
   - **Logo图片** 统一命名为 `[名称]_logo.xxx` (推荐 PNG，要求背景透明，尺寸正方形)。
   - **背景图片** 统一命名为 `[名称]_bg.xxx` (推荐 JPG，横向宽图)。
2. **入库落地流程：**
   - 客户或运营实施人员：首先将整个图片文件夹，通过现有的系统的 `基础数据维护 -> Logo字典 / 背景图字典 / 上传图片` 进行**批量上传**。系统此时会把所有图片的真实 URL 和源文件名称存入底层字典库。
   - Excel导入程序：在导入院校 Excel 的时候，读取 H列 或 I列 的“图片名”（如 `斯坦福大学_logo.png`）。程序不仅不报错，反而会拿着这个文件名**反查刚才上传好的系统字典库**（`find(d => d.name === "斯坦福大学_logo.png")`），如果存在，则直接提取它的上传服务器 URL (`sysUrl`) 绑定给这所大学入库。如果没查到，则警告客户“图片不存在，请先去图库上传”。
