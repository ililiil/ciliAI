# 局部重绘功能错误修复计划

## 问题分析

### 错误日志
```
[error] 改图API调用失败: Error: 改图API请求失败: 500 INTERNAL SERVER ERROR
at editImage (http://localhost:3002/src/utils/volcengine.js:362:12)

at applyEdit (http://localhost:3002/src/views/ProjectDetail.vue?t=1776235067927:1233:12)
```

### API文档要求（来源：火山引擎即梦AI接口文档）
根据接口文档，`jimeng_image2image_dream_inpaint` 接口要求：

1. **输入参数**：
   - `binary_data_base64`: 数组，包含2张图片的base64编码
     - 第一张：原图
     - 第二张：**单通道灰度图** mask图
   - `prompt`: 编辑提示词

2. **Mask图格式要求**：
   - 单通道灰度图（Grayscale）
   - 像素值0（黑色区域）= 保持原样
   - 像素值255（白色区域）= 待重绘区域
   - 不支持带透明通道的图像

### 当前代码实现问题

**前端 (ProjectDetail.vue)**:
- 第1877-1883行：绘制蒙版时使用 `fillStyle = 'rgba(255, 255, 255, 0.5)'`（半透明白色）
- 第1936行：使用 `canvas.toDataURL('image/png')` 导出为PNG格式（带alpha通道）
- 问题：PNG格式包含4个通道（RGBA），但API需要单通道灰度图

**后端 (app.py)**:
- 第1381行：直接转发mask给API，未做任何转换
- 问题：接收到的mask是RGBA格式，但火山引擎API要求单通道灰度图

## 修复方案

### 方案选择
**在前后端转换Mask格式**：在发送API请求前，将RGBA格式的mask转换为单通道灰度图。

### 具体实施步骤

#### 步骤1：在后端添加Mask转换函数（推荐）
**文件**: `d:\fangtang\ciliAI\ciliAI\app.py`

**修改内容**：
1. 在文件顶部添加辅助函数 `convert_rgba_to_grayscale_mask`
2. 在 `inpaint_image` 函数中，调用API前转换mask格式
3. 确保转换后的mask满足：
   - 单通道灰度
   - 白色区域(255) = 重绘区域
   - 黑色区域(0) = 保持区域
