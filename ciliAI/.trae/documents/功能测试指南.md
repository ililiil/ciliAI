## 三、数据库验证

### 3.1 检查 chat_sessions 表
```sql
sqlite3 fangtang.db

-- 查看所有会话
SELECT * FROM chat_sessions ORDER BY update_time DESC;

-- 查看特定项目的会话
SELECT * FROM chat_sessions WHERE project_id = 1;
```

### 3.2 验证数据完整性
```sql
-- 检查会话数量
SELECT COUNT(*) as session_count FROM chat_sessions;

-- 检查会话与项目的关联
SELECT cs.*, p.title as project_title 
FROM chat_sessions cs 
LEFT JOIN projects p ON cs.project_id = p.id;
```

## 四、API 测试

### 4.1 使用浏览器控制台测试

#### 获取会话列表
```javascript
fetch('/api/chat/sessions?invite_code=XXXXXXXX&project_id=1')
  .then(r => r.json())
  .then(console.log)
```

#### 创建会话
```javascript
fetch('/api/chat/sessions', {
  method: 'POST',
  headers: {'Content-Type': 'application/json'},
  body: JSON.stringify({
    invite_code: 'XXXXXXXX',
    project_id: 1,
    title: '测试会话',
    selected_people: 'script'
  })
}).then(r => r.json()).then(console.log)
```

#### 更新会话
```javascript
fetch('/api/chat/sessions/1', {
  method: 'PUT',
  headers: {'Content-Type': 'application/json'},
  body: JSON.stringify({
    invite_code: 'XXXXXXXX',
    title: '更新后的标题',
    conversation_id: 'dify_conversation_123'
  })
}).then(r => r.json()).then(console.log)
```

#### 删除会话
```javascript
fetch('/api/chat/sessions/1?invite_code=XXXXXXXX', {
  method: 'DELETE'
}).then(r => r.json()).then(console.log)
```

#### 获取会话消息
```javascript
fetch('/api/chat/sessions/1/messages?invite_code=XXXXXXXX')
  .then(r => r.json())
  .then(console.log)
```

## 五、常见问题排查

### 5.1 会话列表为空
- 检查邀请码是否正确
- 检查网络请求是否正常
- 查看浏览器控制台是否有错误

### 5.2 无法创建会话
- 检查后端服务是否运行
- 检查数据库权限
- 查看后端日志

### 5.3 历史记录不显示
- 检查 API 响应
- 确认数据是否正确保存到数据库
- 检查前端数据映射是否正确

### 5.4 数据库错误
如果遇到数据库错误，可能需要重启后端服务：
```bash
# 停止服务
# Ctrl+C

# 重启服务
python app.py
```

## 六、性能优化建议

### 6.1 大量数据时的优化
- 会话列表：前端已实现按更新时间排序
- 历史记录：后端支持分页加载
- 图片加载：使用懒加载

### 6.2 数据清理
定期清理旧数据：
```sql
-- 删除30天前的会话（示例）
DELETE FROM chat_sessions 
WHERE update_time < datetime('now', '-30 days');
```
