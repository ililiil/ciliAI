# 漫剧工具选择器修改说明

## ✅ 已完成的修改

### 修改内容
将角色选择器改为**漫剧常用工具选择器**

**修改前**：
```html
<label>选择角色：</label>
<el-select v-model="selectedPeople" placeholder="选择角色" size="small">
  <el-option label="角色 1" value="1" />
  <el-option label="角色 2" value="2" />
  <el-option label="角色 3" value="3" />
</el-select>
```

**修改后**：
```html
<label>选择工具：</label>
<el-select v-model="selectedPeople" placeholder="选择工具" size="small">
  <el-option label="剧本讨论" value="1" />
  <el-option label="角色设定" value="2" />
  <el-option label="场景描写" value="3" />
</el-select>
```

---

## 🎭 漫剧常用工具

| 工具名称 | 对应 people 值 | 说明 |
|---------|---------------|------|
| **剧本讨论** | `'1'` | 用于剧本创作、情节设计、故事结构讨论 |
| **角色设定** | `'2'` | 用于角色性格塑造、人物关系、外貌描写 |
| **场景描写** | `'3'` | 用于场景氛围营造、环境描写、镜头语言 |

---

## 📋 使用示例

### 场景 1：创作剧本
1. 在工具选择器中，选择"**剧本讨论**"（people = '1'）
2. 输入："我想创作一个关于青春校园的故事"
3. AI 会提供剧本创作相关的建议和帮助

### 场景 2：设计角色
1. 在工具选择器中，选择"**角色设定**"（people = '2'）
2. 输入："我需要一个反派角色，性格残忍"
3. AI 会提供角色设定相关的建议和帮助

### 场景 3：描写场景
1. 在工具选择器中，选择"**场景描写**"（people = '3'）
2. 输入："描写一个阴森恐怖的废弃医院"
3. AI 会提供场景描写相关的建议和帮助

## 🔄 工作流对应关系

根据 Dify 工作流配置，`people` 参数控制条件分支：

```yaml
cases:
  - case_id: 'true'
    conditions:
      - comparison_operator: contains
        value: '1'  # 剧本讨论
        variable_selector:
          - '1776059724557'
          - people
  
  - case_id: e3489c3b...
    conditions:
      - comparison_operator: contains
        value: '2'  # 角色设定
        ...
  
  - case_id: a54daa67...
    conditions:
      - comparison_operator: contains
        value: '3'  # 场景描写
        ...
```

### 不同工具的 AI 响应
每个工具会触发不同的工作流分支，提供针对性的专业建议：
- **剧本讨论** → 触发剧本创作专家模式
- **角色设定** → 触发角色设计专家模式
- **场景描写** → 触发场景描写专家模式

## 💡 高级功能

### 1. 切换工具
可以在对话过程中随时切换工具，AI 会根据新选择的工具提供不同的专业建议。

### 2. 多轮对话
每个工具选择后，都支持多轮对话，方便深入讨论：
- 剧本讨论：可以逐步完善剧情结构
- 角色设定：可以逐步丰富角色细节
- 场景描写：可以逐步优化描写细节

### 3. 组合使用
可以先使用"剧本讨论"设计剧情，再切换到"角色设定"设计人物，最后使用"场景描写"完善环境描写。

```vue
  <el-option label="剧本讨论" value="1" />
  <el-option label="角色设定" value="2" />

  <!-- <el-option label="对话编写" value="4" /> -->
  <!-- <el-option label="情感分析" value="5" /> -->

```

**注意**：
- label 是显示给用户的名称
- value 是传递给 Dify API 的值
- 需要同步修改 Dify 工作流的条件分支配置
