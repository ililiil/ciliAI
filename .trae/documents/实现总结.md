# AI 历史数据存储方案 - 实现总结

## 📋 项目概述

### 需求背景
用户需要在"我的项目"中保存 AI 对话和 AI 生图的历史数据，以便在下次登录时能够继续查看和使用。

### 实现目标
1. ✅ 持久化保存 AI 对话会话（会话ID、标题、消息等）
2. ✅ 持久化保存 AI 生图历史记录（提示词、图片、参数等）
3. ✅ 支持跨会话查看历史数据
4. ✅ 实现数据的增删改查操作

---

## 🗄️ 数据库设计

### 新增表：chat_sessions

```sql
CREATE TABLE chat_sessions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    project_id INTEGER,
    user_id INTEGER NOT NULL,
    conversation_id TEXT,           -- Dify API 会话ID
    title TEXT NOT NULL DEFAULT '新对话',  -- 会话标题
    selected_people TEXT DEFAULT 'script',  -- 选中的AI角色
    message_count INTEGER DEFAULT 0,
    create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    update_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (project_id) REFERENCES projects(id) ON DELETE CASCADE,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);
```

### 索引优化
- `idx_chat_sessions_user_id`: 用户ID索引
- `idx_chat_sessions_project_id`: 项目ID索引

## 🌐 API 接口设计

### 会话管理 API

| 方法 | 端点 | 功能 |
|------|------|------|
| GET | `/api/chat/sessions` | 获取会话列表 |
| POST | `/api/chat/sessions` | 创建新会话 |
| PUT | `/api/chat/sessions/<id>` | 更新会话信息 |
| DELETE | `/api/chat/sessions/<id>` | 删除会话 |
| GET | `/api/chat/sessions/<id>/messages` | 获取会话消息 |

### 图片历史 API（扩展）
已有接口：
- `GET /api/records`: 获取图片生成记录

## 📊 数据流程

### AI 对话流程
```
用户进入项目
    ↓
loadHistoryData()
    ↓
loadChatSessions() ← GET /api/chat/sessions
    ↓
显示会话列表 + 加载消息
    ↓
用户发送消息
    ↓
POST /dify-api/chat-messages (已有逻辑)
    ↓
保存消息到 chat_messages 表 (已有逻辑)
    ↓
updateChatSession() ← PUT /api/chat/sessions/<id>
    ↓
刷新显示
```

### AI 生图流程
```
用户进入生图模块
    ↓
loadImageHistory() ← GET /api/records
    ↓
显示历史列表
    ↓
用户生成新图片
    ↓
POST /api/generate (已有逻辑)
    ↓
保存记录到 generation_records 表 (已有逻辑)
    ↓
loadImageHistory() 刷新历史
    ↓
显示新记录
```
